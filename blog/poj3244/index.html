Description

For every pair of triplets, Ta=(Ia,Ja,Ka)T_a = (I_a, J_a, K_a)Ta​=(Ia​,Ja​,Ka​) and Tb=(Ib,Jb,Kb)T_b = (I_b, J_b, K_b)Tb​=(Ib​,Jb​,Kb​), we define the difference value between TaT_aTa​ and TbT_bTb​ as follows:

D(Ta,Tb)=max⁡(Ia−Ib,Ja−Jb,Ka−Kb)−min⁡(Ia−Ib,Ja−Jb,Ka−Kb)D(T_a, T_b) = \max (I_a − I_b, J_a − J_b, K_a − K_b) − \min (I_a − I_b, J_a − J_b, K_a − K_b)D(Ta​,Tb​)=max(Ia​−Ib​,Ja​−Jb​,Ka​−Kb​)−min(Ia​−Ib​,Ja​−Jb​,Ka​−Kb​)

Now you are given NNN triplets, could you write a program to calculate the sum of the difference values between every unordered pair of triplets?

Input

The input consists of several test cases.
Each test case begins with a line containing an integer NNN, denotes the number of triplets. Assume that we number the triplets as T1,T2,…,TNT_1, T_2, \dots , T_NT1​,T2​,…,TN​. Then, there are following NNN lines, each line contains three integers, giving the elements of each triplet.
A case with N=0N = 0N=0 indicates the end of the input.

Sample Input

2
1 2 3
3 2 1
3
1 3 2
4 0 7
2 2 9
0

Sample Output

4
20

Hint

Case 1: D(T1,T2)=4D(T_1,T_2)=4D(T1​,T2​)=4
Case 2: D(T1,T2)+D(T1,T3)+D(T2,T3)=8+8+4=20D(T_1,T_2)+D(T_1,T_3)+D(T_2,T_3)=8+8+4=20D(T1​,T2​)+D(T1​,T3​)+D(T2​,T3​)=8+8+4=20

You can assume that NNN, the number of triplets in each case, will not exceed 200,000 and the elements in triplets fit into [−106,106][-10^6,10^6][−106,106].
The size of the input will not exceed 5 MB.

Analysis

Solutoin #1

很类似上次的HDU 5626 Clarke and points和HDU 6447 YJJ’s Salesman的结合，我们可以先分类讨论（因为这是一个轮换对称式，其实随便考虑一种情况就可以了）。

假设 max⁡(Ia−Ib,Ja−Jb,Ka−Kb)=Ia−Ib\max (I_a − I_b, J_a − J_b, K_a − K_b) = I_a - I_bmax(Ia​−Ib​,Ja​−Jb​,Ka​−Kb​)=Ia​−Ib​，那么就有：

Ia−Ib>Ja−Jb,Ia−Ib>Ka−KbI_a-I_b>J_a-J_b,I_a-I_b>K_a-K_bIa​−Ib​>Ja​−Jb​,Ia​−Ib​>Ka​−Kb​

变换得到：

Ia−Ja>Ib−Jb,Ia−Ka−ib−KbI_a-J_a>I_b-J_b,I_a-K_a-i_b-K_bIa​−Ja​>Ib​−Jb​,Ia​−Ka​−ib​−Kb​

令 Ai=Ii−Ji,Bi=Ii−KiA_i=I_i-J_i,B_i=I_i-K_iAi​=Ii​−Ji​,Bi​=Ii​−Ki​，显然 Ai>Aj,Bi>BjA_i>A_j,B_i>B_jAi​>Aj​,Bi​>Bj​。
这就转换成区间选点问题了。只需要先按照 AiA_iAi​ 排序，然后用线段树或者树状数组维护下就可以了～

Solution #2

上面的解法显然太麻烦了，其实有一种更加精妙的做法。

首先我们要知道一个超级重要的公式：

max⁡(a,b,c)−min⁡(a,b,c)=∣a−b∣+∣b−c∣+∣c−a∣2\max(a,b,c)-\min(a,b,c) = \frac {|a-b|+|b-c|+|c-a|} 2max(a,b,c)−min(a,b,c)=2∣a−b∣+∣b−c∣+∣c−a∣​

所以可以得到：

D(Ta,Tb)=max⁡(Ia−Ib,Ja−Jb,Ka−Kb)−min⁡(Ia−Ib,Ja−Jb,Ka−Kb)=∣Ia−Ib−Ja+Jb∣+∣Ja−Jb−Ka−Kb∣+∣Ka−Kb−Ia+Ib∣2D(T_a, T_b) = \max (I_a − I_b, J_a − J_b, K_a − K_b) − \min (I_a − I_b, J_a − J_b, K_a − K_b) \\\\ = \frac {|I_a-I_b-J_a+J_b| + |J_a-J_b-K_a-K_b| + |K_a-K_b-I_a+I_b|} 2D(Ta​,Tb​)=max(Ia​−Ib​,Ja​−Jb​,Ka​−Kb​)−min(Ia​−Ib​,Ja​−Jb​,Ka​−Kb​)=2∣Ia​−Ib​−Ja​+Jb​∣+∣Ja​−Jb​−Ka​−Kb​∣+∣Ka​−Kb​−Ia​+Ib​∣​

同样令 Ai=Ii−Ji,Bi=Ji−Ki,Ci=Ki−AiA_i=I_i-J_i,B_i=J_i-K_i,C_i=K_i-A_iAi​=Ii​−Ji​,Bi​=Ji​−Ki​,Ci​=Ki​−Ai​，整理得到：

D(Ta,Tb)=∣Aa−Ab∣+∣Ba−Bb∣+∣Ca−Cb∣2D(T_a,T_b)=\frac {|A_a-A_b|+|B_a-B_b|+|C_a-C_b|} 2D(Ta​,Tb​)=2∣Aa​−Ab​∣+∣Ba​−Bb​∣+∣Ca​−Cb​∣​

所以只需要对于每个 AiA_iAi​，累计它与所有小于它的 AiA_iAi​ 之差之和，对于 BiB_iBi​ 和 CiC_iCi​ 同理。即：

∑D(Ti,Tj)=∑Ai<Aj(Aj−Ai)+∑Ci<Cj(Cj−Ci)+∑Ci<Cj(Cj−Ci)2\sum D(T_i,T_j) = \frac {\sum_{A_i< A_j} (A_j-A_i) +\sum_{C_i< C_j} (C_j-C_i) + \sum_{C_i< C_j} (C_j-C_i) } {2}∑D(Ti​,Tj​)=2∑Ai​<Aj​​(Aj​−Ai​)+∑Ci​<Cj​​(Cj​−Ci​)+∑Ci​<Cj​​(Cj​−Ci​)​

这个很好做，维护下加和就好了。

嗯，原来在 FAQ 里面说了：

Prior to the compiler upgrade in May 2009, we serviced C and C++ submissions using GCC 3.4.2. That version of GCC relies on the old MS VC++ 6 runtime library, which does not support the %lld and %llu specifiers for signed and unsigned long long types but allows the standard-incompliant %lf and %lg specifiers for floating-point types. The new GCC 4.4.0 comes with its own ISO C-conformant implementation of the printf function. Now you can use %lld and %llu with either C/C++ compiler, but %lf and %lg only work with MS VC++ 2008 Express. As a rule of thumb, you should always use the %f and %g specifiers instead for floating-point types.

…可怕……

#include<cstdio>
#include<cstring>
#include<iostream>
#include<algorithm>
using namespace std;
const long long maxn=200005;
long long n,a[maxn],b[maxn],c[maxn];
long long ans=0;

int main(){
    for (;;){
        scanf("%lld",&n);
        if (n==0) break;
        ans=0;

        for (long long i=1;i<=n;i++){
            long long x,y,z;
            scanf("%lld%lld%lld",&x,&y,&z);
            a[i]=x-y,b[i]=y-z,c[i]=z-x;
        }
        sort(a+1,a+1+n);sort(b+1,b+1+n);sort(c+1,c+1+n);

        long long sum_a=0,sum_b=0,sum_c=0;
        for (long long i=1;i<=n;i++){
            ans+=a[i]*(i-1)-sum_a;
            ans+=b[i]*(i-1)-sum_b;
            ans+=c[i]*(i-1)-sum_c;
            sum_a+=a[i];sum_b+=b[i];sum_c+=c[i];
        }
        printf("%lld\n",ans/2);
    }
    return 0;
}
